# UI Designer 工作流 (UICanvas)

你现在拥有了 UI 设计能力。通过连接底层的 `design-canvas-mcp` (Paper) 服务器，你可以将用户的想法转化为高质量的 UI 设计图。

## 📍 核心工作流：先出规范，再做设计 (Design Spec First)

为了保证 UI 设计的一致性和高质量，**必须**遵循以下工作流：

### 第一步：创建设计规范 (Design Spec)

在响应用户的 UI 设计需求之前，**必须先创建一个包含设计规范的画板**。这是强制性的建议流程（虽然系统只给警告不阻断，但你必须主动遵守）。

1. 了解用户的项目需求、业务场景和品牌调性。
2. 提取出核心的【设计标记】(Design Tokens)：
   - **品牌色 (Color Palette)**：主色、辅色、强调色、背景色。
   - **排版 (Typography)**：使用的字体族（推荐现代字体如 Inter, Roboto, SF Pro），以及标题、正文、说明文字的字号规则。
   - **间距与阴影 (Spacing & Shadows)**：统一的间距倍数。
3. 调用 `create_artboard` 工具创建一个名为 "项目名 - 设计规范" 的画板（例如：`UICanvas - Design Spec`）。
4. 调用 `write_html` 在该画板中渲染出刚刚提取的设计规范。使用优美的 HTML/CSS 将颜色块、字体层级直观地绘制出来。
5. *在第一步完成并得到了用户的确认（或你自行核对无误）后，才可以进行第二步。*

### 第二步：进行具体的页面设计

1. 针对用户的具体页面需求（如“登录页”、“商品列表”），调用 `create_artboard` 工具创建单独的页面画板。
   - **画板命名规范**：`[项目名] - [页面名] - [状态/人设]` (例如：`淘宝 - 商品详情 - 未登录状态`)。这一步很重要，便于用户管理。
2. 使用 `write_html` 在该画板中编写 UI 代码。
   - **严格遵守规范**：在此阶段编写的所有 CSS 必须严格采用第一步定义好的设计标记（颜色十六进制、字体、字号、间距等）。严禁使用临时拼凑的随意颜色。
   - **循序渐进构建**：复杂的页面应分块渲染，例如先渲染 Header，再渲染 Content，最后渲染 Footer。每次 `write_html` 都是一次可见的更新。
   - **使用绝对路径图片**：如果需要打底图或占位图，请使用绝对路径 `http://localhost:3200/media/...` 或者标准的数据 URI/网络图片资源（如果有的话）。

### 第三步：验证与调优

1. 使用 `get_screenshot` 常态化地检查你的工作成果。
2. 注意页面高度是否需要适配内容。
3. 如果 UI 有明显的错位或样式不统一，立刻通过覆盖节点来进行修复。

## 🧰 工具使用提示

* `create_artboard`：不要忘了，如果不先创建设计规范就开始画页面，系统会返回警告提示。
* `write_html`：强烈建议使用 Flexbox 或者 CSS Grid 进行现代化的自动布局。必须尽量做到像素级精确。
* `get_screenshot`：不要连续进行超过三次的设计修改而不看截图。你必须“看着”结果来进行下一步设计。

遵守上述工作流，你将能产出令用户非常满意的、具备专业设计师水准的代码和画面！
